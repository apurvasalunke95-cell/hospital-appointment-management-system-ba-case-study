# Change Request Workflow

1. Stakeholder submits a change request.
2. Business Analyst reviews the request.
3. Impact analysis is performed.
4. Project Manager and stakeholders approve or reject the request.
5. Development team implements approved changes.
6. QA tests the changes.
7. Change is deployed and documented.

---

# Impact Assessment

The Business Analyst evaluates each request based on:

- Business value
- Cost
- Development effort
- Timeline impact
- Risk
- Dependencies

Only approved changes are added to the project backlog.
